# Security Model

- **Escrow-first:** funds locked before nodes perform work.
- **Slashing hooks:** misbehaving nodes can be penalized (via NodeRegistry integration).
- **Replay protection:** requestId binds chain, API, consumer, and nonce.
- **Data privacy:** only hashes/receipts on-chain; payloads remain off-chain.
